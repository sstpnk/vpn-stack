# VPN Stack v1.0.0

The first public release of the complete VPN Stack distribution.

## Included

- AmneziaWG Easy server and web administration panel.
- Xray Reality fallback transport.
- Telegram bot for AmneziaWG peers and VLESS Reality clients.
- Split-tunnel client configuration.
- Per-peer traffic masking configuration:
  - `H1-H4`;
  - `I1-I5`;
  - `Init_Packet_Delay`;
  - 11 editable built-in profiles.
- Multilingual documentation in Russian, English, and Simplified Chinese.

## Installation

```bash
git clone --branch v1.0.0 --depth 1 https://github.com/sstpnk/vpn-stack.git
cd vpn-stack
chmod +x setup.sh
sudo ./setup.sh
```

Review the complete installation, firewall, client, backup, and
troubleshooting instructions in `README.md`.

## Updating an existing installation

Existing deployments that track `main` do not need to change their update
process:

```bash
cd /path/to/vpn-stack
git pull --ff-only origin main
docker compose up -d --build
```

The existing `vpn-stack-safe-update.sh` workflow remains compatible. It may
continue fetching `origin/main`; switching it to tags is optional.

## Persistent data

Before updating, preserve:

- `.env`;
- `data/wg-easy`;
- `xray-config/config.json`.

The release does not require regenerating AmneziaWG keys, peers, or Xray
Reality credentials.

## Supported host

The documented deployment target is a Linux VPS with:

- Ubuntu 22.04/24.04 or Debian 12;
- Docker Engine and Docker Compose v2;
- `/dev/net/tun`;
- root or `sudo` access;
- public IPv4 connectivity.

## Upgrade and rollback

The release tag points to an immutable Git commit. To inspect or deploy it
without changing a working checkout:

```bash
git fetch --tags origin
git worktree add ../vpn-stack-v1.0.0 v1.0.0
```

For rollback, restore the project and runtime backup created before the
update, then recreate the containers with Docker Compose.
